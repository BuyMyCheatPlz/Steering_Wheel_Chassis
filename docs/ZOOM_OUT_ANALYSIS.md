## 1. 系统概览 — 三层架构拓扑

```
┌──────────────────────────────────────────────────────────────────┐
│                     Layer 3: 任务调度层 (FreeRTOS)                │
│  ┌─────────────┐   ┌─────────────┐   ┌─────────────┐            │
│  │ 遥控处理任务  │   │ 运动控制任务  │   │ 调试发送任务  │            │
│  │ 事件驱动     │   │ 500Hz 周期  │   │ 100Hz 周期  │            │
│  │ Signal_Queue ├──→│ Signal_Queue│   │ 只读调试数据 │            │
│  └─────────────┘   └──────┬──────┘   └─────────────┘            │
├────────────────────────────┼──────────────────────────────────────┤
│                     Layer 2: 控制核心层                           │

│  │  └──────────┘ └──────────┘ └──────────┘ └──────────┘      │  │
│  │       ↓                                      ↓              │  │
│  │  ┌──────────┐                           ┌──────────┐       │  │
│  │  │急停检测   │                           │CAN 发送  │       │  │
│  │  │fail_count│                           │8路电流   │       │  │

├──────────────────────────────────────────────────────────────────┤
│                     Layer 1: 硬件抽象层                           │
│  ┌───────┐ ┌───────┐ ┌───────────┐ ┌───────┐ ┌───────────┐     │

│     CAN1 / UART2 / UART4 / GPIO / DMA / FreeRTOS CMSIS_V2      │
└──────────────────────────────────────────────────────────────────┘
```

**关键架构观察:**

1. **层间耦合度低:** 任务层通过 `Signal_Queue` (FreeRTOS 队列) 与控制层通信，控制层通过硬件驱动 API 访问电机/编码器。层间接口清晰。
2. **控制环路闭合在 Layer 2:** `ControlLoop()` 内完成"读取→解算→PID→发送"的完整闭环，不依赖任务调度时序，只要被周期性调用即可工作。
3. **硬件访问无任务冲突:** 编码器 (4 组软 I2C) 和 CAN 发送都在 `Movement_Handle` 任务上下文中执行，不存在多任务竞争硬件总线的风险。

## 2. 数据流拓扑 — 从遥控器到电机

```
遥控器 (DJI DT7/DR16)
    │  18 字节 DBUS 帧, 7ms 帧间隔
    │  UART2 DMA CIRCULAR 双缓冲接收
    ▼
HAL_UARTEx_RxEventCallback (ISR)
    │  帧头 0xA0 校验 → RemoteControl_IdleCallback
    │  → 解析原始值 + 归一化 [-1, +1]
    │  → 写入 cached_state (18 字节, P0-2 临界区保护)
    │  → 释放 DbusReady 信号量 (FromISR)
    ▼
Receive_and_Process_Signal (任务)
    │  等待 DbusReady 信号量
    │  → 读 cached_state (临界区)
    │  → LossProtectionTick() — 失联保护状态机
    │  → 映射: ly→vx, lx→vy, ry→wz
    │  → 写入 Signal_Queue
    ▼
Motor_Movement (任务, 500Hz)
    │  等待 2ms 周期
    │  → 从 Signal_Queue 取 Vcmd {vx, vy, wz}
    │  → SteeringChassis_SetVelocity(vx, vy, wz)
    │  → SteeringChassis_ControlLoop()
    ▼
ControlLoop 内部 (顺序执行, 无 RTOS 阻塞)
    │
    │  ① ReadEncoders()        — 4 路 MT6701 软 I2C 并发读取
    │  ② ReadMotorFeedback()   — 8 路 M3508 CAN 反馈 (前一周期缓存)
    │  ③ InverseKinematics()   — Vcmd → 4轮目标舵角 + 驱动转速
    │      ├─ 最短路径 + 施密特滞回 (P1-5)
    │      └─ 减速比补偿 (P2-9)
    │  ④ ComputeSteerPID(idx)  — 串级: 位置环→速度环→电流 (4轮 × 独立实例)
    │  ⑤ ComputeDrivePID(idx)  — 单速度环→电流 (4轮 × 独立实例)
    │  ⑥ ApplyCurrentRamp(idx) — 电流变化率限幅 (±4000mA/周期)
    │  ⑦ SendMotorCurrents()   — CAN 批量发送 8 路电流指令
    ▼
M3508 电机执行 (CAN 2ms 反馈周期)
```

**数据流关键特性:**

| 特性 | 说明 |
|------|------|
| **控制周期** | 500Hz (2ms), 与 CAN 反馈周期对齐 |
| **反馈延迟** | 1 个周期 (当前周期使用上一周期 CAN 反馈) |
| **遥控延迟** | 事件驱动 (7ms DBUS + 2ms 任务调度) |
| **队列深度** | Signal_Queue 默认深度, 可缓冲约 2-3 帧 |
| **状态拷贝** | P0-2 临界区保护 18 字节 atomic read |

## 3. 状态机全景 — 4 状态生命周期

```
                    ┌─────────────────────────────────┐
                    │         CHASSIS_HOMING            │
                    │  上电回正 (编码器健康检查隐含)      │
                    │  舵向→位置PID, 驱动→静止           │
                    └──────────────┬──────────────────┘
                                   │ 4轮全部到位 (|Δθ| < 3°)
                                   │ → PID_Reset(all)
                                   ▼
                    ┌─────────────────────────────────┐
                    │         CHASSIS_NORMAL            │
                    │  正常控制 (运动学→PID→CAN)        │
                    └───┬─────────────────────┬───────┘
                        │                     │
         编码器连续失败   │                     │ SW1=DOWN 或
         ≥ 10次          │                     │ 遥控器失联
                        ▼                     ▼
                    ┌─────────────────────────────────┐
                    │         CHASSIS_ESTOP             │
                    │  急停 (电流=0, PID_Reset)         │
                    └───┬─────────────────────────────┘
                        │ SW1=UP 确认恢复
                        │ → 进入恢复检查
                        ▼
                    ┌─────────────────────────────────┐
                    │    CHASSIS_ESTOP_RECOVER         │
                    │  编码器健康检查 (连续5次)          │
                    │  → SteeringChassis_IsEncoderHealthy()
                    │  达标 → 重新回正 → NORMAL         │
                    └─────────────────────────────────┘
```

**状态转换触发条件:**

| 转换 | 触发源 | 延迟 | 可逆 |
|------|--------|------|------|
| HOMING → NORMAL | 4 轮到达目标角度 | 0 (立即) | ❌ (不可逆) |
| NORMAL → ESTOP | 编码器故障 / SW1=DOWN / 失联 | 0 (立即) | ✅ (经由 RECOVER) |
| ESTOP → RECOVER | SW1=UP (遥控层确认) | 0 | ❌ (单向) |
| RECOVER → HOMING | 编码器 5 次连续健康 | ~500ms | ❌ (单向) |

**状态安全性分析:**

1. **HOMING 是唯一入口状态:** 上电始终从回正开始，确保初始角度已知。
2. **ESTOP 是安全汇点:** 任何异常路径最终收敛到 ESTOP (电流=0)。
3. **RECOVER 是单向通道:** ESTOP → RECOVER → HOMING → NORMAL，不允许直接跳回 NORMAL，强制重新校准。
4. **无自旋状态:** 所有状态都有明确的到达路径，不存在死锁。

### 4.2 故障覆盖率矩阵

| 故障类型 | 检测机制 | 响应时间 | 响应动作 | 覆盖率 |
|----------|----------|----------|----------|--------|
| 遥控器断电 | DBUS 帧超时 (50ms) | ~7ms (P1-3修复) | 急停→关断 | ✅ 100% |
| 遥控器信号干扰 | 帧头校验 + 超时 | ~7ms | 忽略坏帧, 超时急停 | ✅ 100% |
| 编码器物理断开 | I2C 连续读取失败 | ~100ms (10次) | 急停 | ⚠️ 依赖 I2C 返回错误 |
| 编码器读值跳变 | ❌ 未检测 | N/A | N/A | ❌ 0% (见 4.3) |
| CAN 总线断开 | M3508 反馈缺失 | 2ms | 反馈不可用但 PID 继续计算 | ⚠️ 部分覆盖 |
| 电机堵转 | ❌ 未检测 | N/A | N/A | ❌ 0% |
| Stack 溢出 | FreeRTOS 检测 | 运行时 | 硬故障 | ✅ 100% |
| DMA 缓冲区溢出 | DBUS 帧解析丢弃 | <1ms | 丢弃超长帧 | ✅ 100% |

### 4.3 未覆盖的风险

#### 4.3.1 编码器跳变检测缺失 (新风险)

**风险描述:** MT6701 可能因磁场干扰或接触不良产生跳变读数 (例如从 45° 跳到 90°)，目前无异常跳变检测。

**危害:** 跳变角度被 PID 当作位置误差 → 舵向电机剧烈转动 → 机械损坏。

**建议缓解:**
```c
// 在 ReadEncoders() 中增加跳变检测
#define ENCODER_JUMP_THRESHOLD_DEG  30.0f  // 单周期跳变超过 30° 视为异常

static float g_last_valid_angle[4];  // 上一周期有效角度

// 在 encoder_valid 后:
float delta = fabsf(g_wheel[i].encoder_angle - g_last_valid_angle[i]);
if (delta > ENCODER_JUMP_THRESHOLD_DEG) {
    g_wheel[i].encoder_valid = 0;  // 标记无效
    g_wheel[i].encoder_fail_count++;
    // 不更新 g_last_valid_angle
} else {
    g_last_valid_angle[i] = g_wheel[i].encoder_angle;
}
```

**优先级:** P1 (应在下一轮审查中修复)

#### 4.3.2 电机堵转保护缺失

**风险描述:** 如果舵向电机卡死 (机械限位、异物)，电流将持续输出上限值但角度不变。

**危害:** 电机发热 → 烧毁驱动/电机。

**建议缓解:**
- 监测 `steer_pos_error` 持续 > 阈值且 `steer_rpm` ≈ 0
- 超时自动急停并标记故障

**优先级:** P1

#### 4.3.3 CAN 发送失败检测缺失

**风险描述:** `M3508_SendAll()` 使用 `HAL_CAN_AddTxMessage`，当前未检查返回值。如果 CAN 发送邮箱满 (3 个邮箱全部 pending)，指令将静默丢失。

**危害:** 电机在急停时可能仍然转动 (因为旧电流指令未更新)。

**建议缓解:**
- 检查 `HAL_CAN_AddTxMessage` 返回值
- 失败时尝试 `HAL_CAN_AbortTxRequest` 清理最老邮箱

**优先级:** P2

## 5. 同步边界分析

### 5.1 中断/Task 数据共享

```
┌──────────────────────────────────────────────────────────────┐
│                    ISR 上下文                                │
│  HAL_UARTEx_RxEventCallback (USART2 IDLE)                   │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  RemoteControl_IdleCallback()                        │   │
│  │    → 扫描 dbus_rx_buffer (DMA 缓冲区)                │   │
│  │    → 解析 18 字节帧                                  │   │
│  │    → 写入 cached_state (18 字节)      ← ⚠️ 并发写入点 │   │
│  │    → xSemaphoreGiveFromISR (DbusReady)               │   │
│  └──────────────────────────────────────────────────────┘   │
│  ↑ DMA 完成中断 (DMA 写 dbus_rx_buffer, ISR 读)              │
│  ↑ DBUS 帧接收 ~7ms 间隔                                    │
├──────────────────────────────────────────────────────────────┤
│                    Task 上下文                               │
│  Receive_and_Process_Signal (Low Priority)                   │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  osSemaphoreAcquire (DbusReady)                      │   │
│  │    → RemoteControl_GetState()                        │   │
│  │    → __disable_irq() / memcpy / __enable_irq()       │   │
│  │      ← P0-2 临界区保护: 原子读 cached_state          │   │
│  │    → LossProtectionTick()  ← ✅ 移出 if(connected)   │   │
│  │    → 写入 Signal_Queue (FreeRTOS 队列, 线程安全)      │   │
│  └──────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────┐
│                    Task 上下文                               │
│  Motor_Movement (Low Priority, 500Hz)                        │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  从 Signal_Queue 读取 Vcmd                            │   │
│  │  SteeringChassis_ControlLoop()                       │   │
│  │    → ReadEncoders()  — 仅读硬件 (I2C)                │   │
│  │    → ReadMotorFeedback() — 仅读 g_m3508_fb[]         │   │
│  │    → 计算 → CAN 发送                                 │   │
│  └──────────────────────────────────────────────────────┘   │
│  ↑ 所有硬件访问都在此 Task 上下文中 (无 ISR 竞争)            │
└──────────────────────────────────────────────────────────────┘
```

### 5.2 关键同步点评估

| 同步点 | 数据结构 | 并发访问 | 保护机制 | 安全 |
|--------|----------|----------|----------|------|
| ISR → Task | `cached_state` (18B) | ISR 写 + Task 读 | P0-2 临界区 `__disable_irq()` | ✅ |
| ISR → Task | `dbus_rx_buffer` | DMA 写 + ISR 读 | UART IDLE 确保 DMA 静默 | ✅ |
| Task → Task | `Signal_Queue` | 遥控写 + 运动读 | FreeRTOS 队列 (CMSIS_V2) | ✅ |
| Task 内部 | `g_wheel[4]` | 仅运动任务访问 | 无并发 | ✅ |
| Task 内部 | `g_steer_pos_pid[4]` | 仅运动任务访问 | 无并发 | ✅ |
| CAN ISR → Task | `g_m3508_fb[8]` | ISR 写, Task 读 | ❌ 无保护! | ⚠️ (见 5.3) |

### 5.3 CAN 反馈的竞态风险 (新发现)

`g_m3508_fb[8]` 在 `HAL_CAN_RxFifo0MsgPendingCallback` (ISR) 中写入，在 `ReadMotorFeedback()` (Task) 中读取，但 `M3508_Feedback` 结构体可能大于 CPU 字长 (假设 12 字节: rpm ×4 + current ×2 + angle ×6)，在 Cortex-M4 上非原子读取。

**危害:** Task 可能读到半更新的反馈数据 (例如 `rpm` 是新值但 `current` 是旧值)。

**建议缓解:**
- 在 `M3508_GetFeedback()` 中增加临界区保护 (类似 P0-2)
- 或使用 `uint8_t feedback_seq` 序列号检测撕裂

**优先级:** P1

## 6. 时序分析

### 6.1 控制环路时间预算 (500Hz = 2000μs)

| 阶段 | 操作 | 估算时间 | 占比 |
|------|------|----------|------|
| ① | ReadEncoders (4 × 软 I2C) | ~800μs (200μs/路) | 40% |
| ② | ReadMotorFeedback (8 路结构体拷贝) | ~10μs | 0.5% |
| ③ | InverseKinematics (浮点三角运算) | ~200μs | 10% |
| ④ | ComputeSteerPID ×4 (串级 PID) | ~120μs | 6% |
| ⑤ | ComputeDrivePID ×4 (速度环) | ~40μs | 2% |
| ⑥ | ApplyCurrentRamp ×4 | ~20μs | 1% |
| ⑦ | SendMotorCurrents (4 × CAN 帧) | ~200μs | 10% |
| **总计** | | **~1390μs** | **69.5%** |
| **剩余** | | **~610μs** | **30.5%** |

**结论:** 500Hz 控制周期有 30% 余量，安全。

### 6.2 端到端延迟链

```
遥控器摇杆移动 → DBUS 发送 (7ms)
    → UART DMA 接收 + IDLE 中断 (0.1ms)
    → 信号量释放 + 任务调度 (0~2ms)
    → Receive_and_Process_Signal (0.1ms)
    → Signal_Queue 入队 (0.05ms)
    → Motor_Movement 取队列 (0~2ms)
    → ControlLoop (1.4ms)
    → CAN 发送到电机 (0.2ms)
    
总计延迟: 8.9 ~ 12.9ms
```

对于手动遥控底盘, <15ms 的端到端延迟是可接受的。
